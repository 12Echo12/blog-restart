今日份（基本上都是network相关的，cause自己确实计算机基础知识太薄弱了，最近在狂补）：

  • 网络请求和响应的本质是 电信号 和 光信号

  • 浏览器发起的请求不是自己发给服务器的，是委托操作系统发的

  • 链接不只是最常见的 http 开头的（再次暴露了自己的’浅薄’哈哈哈哈）根据协议和对应处理服务器类型不同可分为：

    - `http:` 使用 HTTP 协议/超文本传输协议/访问 web 服务器
    • ftp: 使用 FTP 协议/文件传输协议/上传下载文件 - file: 指向计算机本地的文件 - mailto: 发送电子邮件

  • URL vs URI

    - `URL` (uniform resource locator统一资源定位符) eg: https://hhsdhh.com/blog/index.html
    • URI (uniform resource identifier统一资源标识符) eg: blog/index.html - 不难看出，URL 里面包含 URI

  • HTTP 主要方法：

    - GET
- POST
- HEAD  ->　只返回消息头
- OPTIONS
- PUT　-> 替换服务器上的文件
- DELETE  -> 删除服务器上的文件
- TRACE
- CONNECT

    - `jpg/jpeg` 
- `png`
- `gif`
