肯定是有什么插件帮我干了这件事儿，于是去搜寻了一番，果然，我司用的就是postcss-px-to-viewport，这个插件会根据我们配置文件中配置的视口宽度，把我们项目里 css 文件中的所有 px 单位转为 vw。了解了以后又去review了一下各种单位还有各种像素的区别，这下算是清晰了很多…

px vw/vh em/rem %

  • px 对应的是 css 像素（逻辑像素），在不同型号的设备上，1px 对应的大小会因为分辨率、像素比、设备像素（物理像素）的不同而不同

  • vw/vh相对单位，分别相对于视口的宽度和高度。以 vw 为例，会把视口的宽度分为 100 份，那 20vw 就是 20% 视口宽度的长度。

  • em/rem 相对单位，都是相对于字体大小。em 相对于当前盒子文本的字体大小，如果没指定，就默认是 16px。rem 相对于 HTML 根元素的字体大小，没指定也是 16px ，但因为 16px 不太好转换，所以我们可以指定 font-size: 62.5% ，这样 font-size 就是 10px ，那相对于 10px 转换时计算量就少很多。

  • % 相对单位，相对于父元素高度。for 普通盒子，就是相对于父盒子、for absolute定位的盒子，就是相对于最近的已定位的祖(先)盒子、for fixed定位的盒子，就是相对于 viewport视口。

设备独立像素 vs css像素（逻辑像素）vs 设备像素（物理像素）

  • 设备独立像素 是一种抽象的度量单位。如 CSS 中的 ‘px’ 、Android中的’dp’、iOS中的’pt’等。

  • css像素 又称逻辑像素，1px 就是一个 css 像素，但是 1个 css 像素在不同设备上的呈现大小又是不同的。因为渲染时逻辑像素会被转化为物理像素，而转换关系是不同的。

  • 物理像素 又称设备像素，指设备能控制显示的最小物理单位。不同设备上不同。

  • 设备像素比(dpr) 物理像素到逻辑像素的转换关系 = 物理像素/逻辑像素

  • 分辨率 通常表示为 m * n，指的是设备屏幕上的水平像素数 * 垂直像素数

  • 每英寸像素数ppi 表示每英寸的像素数目。在移动端中的标准屏幕(160ppi)下 1 个独立设备像素 = 1 个设备像素
