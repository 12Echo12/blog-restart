变量声明

var 、let 、const

(1) 因为 var 有变量提升，所以 var 声明的变量可以重复声明，都会被提升成一个添加到全局 windows 对象中。

(2) const 声明的变量不可修改指的是变量指向的内存地址不能改动。

  • 对于原始数值类型，它的值就保存在变量指向的内存地址中，eg：const a = 2，那 a 的值就是常量 2 ，不可以改变，如果 a = 3 就会报错。
  • 对于复杂数据类型，变量指向的内存地址中存的是该值的地址，那就是该地址不能改变，而不是该引用对象不能改变。eg：const obj = {a : 1} ,obj = {a : 2} 就会报错，但是 obj.a = 2 就不会。

变量提升

造成变量提升的本质原因是执行上下文的创建。js引擎在代码执行前有一个解析的过程，创建了执行上下文，变量提升可以理解为将变量添加到其作用域下的执行上下文中的活动对象中，并初始化为 undefined。

由于let 没有变量提升，所以会有暂时性死区 （即在let 声明之前的执行瞬间被称为暂时性死区，在此阶段引用后面声明的变量都会报错）。

(1) 变量提升的优点：

  • 提升性能，可以为提前为变量分配栈空间！

(2) 变量提升可能会带来的问题：

  • 导致内存泄漏（意外声明全局变量）
  • 出现意外的错误
